# Machine Learning Workflow

A typical machine learning project follows these steps:

```
Problem Definition
        ↓
Data Collection
        ↓
Data Preprocessing
        ↓
Feature Engineering
        ↓
Train/Test Split
        ↓
Model Training
        ↓
Model Evaluation
        ↓
Deployment
        ↓
Monitoring
```

## 1. Problem Definition

Clearly define the problem and expected output.

Example:

- Predict House Prices
- Detect Spam Emails

---

## 2. Data Collection

Gather relevant and high-quality data.

Sources:

- APIs
- Databases
- Public Datasets
- Sensors

---

## 3. Data Preprocessing

Prepare data by:

- Handling missing values
- Removing duplicates
- Encoding categorical variables
- Scaling features

---

## 4. Feature Engineering

Create or select meaningful features to improve model performance.

---

## 5. Model Training

Train a machine learning algorithm using training data.

---

## 6. Model Evaluation

Evaluate using suitable metrics such as Accuracy or MSE.

---

## 7. Deployment

Deploy the trained model as an API or application.

---

## 8. Monitoring

Monitor model performance and retrain when needed.

---

## Key Takeaways

- Data quality significantly impacts model performance.
- Deployment is not the final step—models require continuous monitoring.
